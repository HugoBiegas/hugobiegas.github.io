Ce dépôt contient le code source de mon portfolio professionnel, présentant mes compétences, projets et expériences en tant que **développeur web, mobile et spécialiste en cybersécurité**. Le site est hébergé sur **GitHub Pages** et accessible à l'adresse [hugobiegas.github.io](https://hugobiegas.github.io).

## 📋 Aperçu

Ce portfolio a été entièrement développé **from scratch** en utilisant des technologies web modernes pour offrir une expérience utilisateur optimale. Il met en valeur mon expertise technique à travers une interface élégante, performante et accessible.

## 🚀 Installation et Utilisation

### Prérequis
- Navigateur web moderne (Chrome, Firefox, Safari, Edge)
- Serveur web local (optionnel pour le développement)
- Compte Firebase (optionnel, pour le formulaire de contact)

### Installation Locale

1. **Cloner le dépôt**
   ```bash
   git clone https://github.com/HugoBiegas/hugobiegas.github.io.git
   cd hugobiegas.github.io
   ```

2. **Lancement en local**
   
   **Option 1 : Ouverture directe**
   ```bash
   # Ouvrir index.html dans votre navigateur
   open index.html  # macOS
   start index.html # Windows
   xdg-open index.html # Linux
   ```
   
   **Option 2 : Serveur HTTP local**
   ```bash
   # Avec Python 3
   python -m http.server 8000
   
   # Avec Node.js (http-server)
   npx http-server
   
   # Avec PHP
   php -S localhost:8000
   ```
   
   Puis ouvrir [http://localhost:8000](http://localhost:8000)

### Déploiement

#### GitHub Pages (Recommandé)
1. Fork ce dépôt sur votre compte GitHub
2. Renommer en `votre-username.github.io`
3. Activer GitHub Pages dans les paramètres
4. Le site sera accessible à `https://votre-username.github.io`

#### Autres hébergeurs
- **Netlify** : Drag & drop du dossier ou connexion Git
- **Vercel** : Import depuis GitHub
- **Surge** : `npm install -g surge && surge`

### Configuration du Formulaire de Contact

Pour activer le formulaire de contact, vous devez configurer Firebase :

1. **Créer un projet Firebase**
   - Aller sur [Firebase Console](https://console.firebase.google.com/)
   - Créer un nouveau projet
   - Activer Firestore Database

2. **Configuration**
   ```javascript
   // Dans contact.js, remplacer la configuration Firebase
   const firebaseConfig = {
     apiKey: "votre-api-key",
     authDomain: "votre-projet.firebaseapp.com",
     projectId: "votre-projet-id",
     // ... autres paramètres
   };
   ```

3. **Règles de sécurité Firestore**
   ```javascript
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /contacts/{document} {
         allow create: if request.auth == null 
           && request.resource.data.keys().hasAll(['name', 'email', 'subject', 'message'])
           && request.resource.data.size() == 5; // incluant timestamp
       }
     }
   }
   ```

   ```javascript
   // Modifier assets/js/projects-data.js
   const projectsData = [
   ```

   ```javascript
   // Modifier assets/js/skills.js
   const skillLevels = {
   ```

5. **Couleurs et thème**
   ```css
   /* Modifier assets/css/main.css */
   :root {
     --primary-color: #votre-couleur;
     --secondary-color: #votre-couleur-secondaire;
     /* ... autres variables */
   }
   ```

### Ajouter de Nouvelles Langues

1. **Créer le fichier de traduction**
   ```javascript
   // assets/js/i18n/es.js (exemple pour l'espagnol)
   window.translations = window.translations || {};
   window.translations.es = {
     "page.title.home": "Inicio | Tu Nombre",
     // ... autres traductions
   };
   ```

2. **Mettre à jour le sélecteur**
   ```javascript
   // Dans assets/js/i18n/translations.js
   // Ajouter 'es' aux langues supportées
   ```

## 🏗️ Architecture & Bonnes Pratiques

### 📐 Architecture du Code
- **Modularité** : Séparation claire entre HTML, CSS et JavaScript
- **Composants réutilisables** : Classes CSS et fonctions JS modulaires
- **Nomenclature cohérente** : Convention de nommage BEM pour le CSS
- **Structure sémantique** : HTML5 sémantique pour l'accessibilité et le SEO

### 🔧 Bonnes Pratiques Implémentées
- **Performance** 
  - Optimisation des ressources et temps de chargement
  - Lazy loading des images
  - Minification CSS et compression des images
  - Cache browser optimisé
  
- **Sécurité**
  - Protection contre les vulnérabilités web courantes (XSS, CSRF)
  - Validation côté client et règles Firebase
  - Échappement des données utilisateur
  - Configuration HTTPS

- **Accessibilité (WCAG 2.1)**
  - Navigation au clavier
  - Attributs ARIA appropriés
  - Contraste de couleurs suffisant
  - Support des lecteurs d'écran
  - Focus visible et logique

- **SEO**
  - Structure sémantique HTML5
  - Balises méta optimisées
  - URLs propres et descriptives
  - Schema.org pour les données structurées
  - Sitemap et robots.txt

### 🎯 Patterns de Développement
- **Progressive Enhancement** : Fonctionnalités de base sans JavaScript
- **Mobile-First** : Conception responsive mobile d'abord
